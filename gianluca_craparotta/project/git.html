Introduzione a Git

Cosa è Git?

Git è un sistema di controllo versione distribuito utilizzato per tenere traccia delle modifiche ai file di un progetto software.

Come funziona Git?

Git registra le modifiche ai file del progetto creando una serie di "snapshot" (commit) nel tempo. Ogni commit contiene le modifiche apportate ai file e un riferimento al commit precedente, creando una cronologia delle modifiche.

Principali concetti di Git:

  • Repository: Una collezione di file e cartelle gestiti da Git.
  • Commit: Un'istanza di una serie di modifiche ai file del repository, registrata da Git.
  • Branch: Una linea di sviluppo separata all'interno del repository, che consente di lavorare su funzionalità isolate senza influenzare il codice principale.
  • Merge: L'operazione di unire le modifiche da un branch a un altro, combinando i cambiamenti in un'unica linea di sviluppo.
  • Pull: Il processo di recuperare e integrare le modifiche da un repository remoto nel repository locale.
  • Push: Il processo di caricare le modifiche dal repository locale a un repository remoto.

Perché usare Git?

Git facilita la collaborazione tra sviluppatori, tenendo traccia delle modifiche, consentendo il lavoro su funzionalità separate e facilitando il ripristino delle versioni precedenti del codice.
