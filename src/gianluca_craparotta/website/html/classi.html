Programmazione Orientata agli Oggetti (OOP)

La programmazione orientata agli oggetti (OOP, Object-Oriented Programming) è un paradigma di programmazione che utilizza oggetti e le loro interazioni per progettare e programmare applicazioni. I principali concetti dell'OOP sono:

1. Classe

Una classe è un modello per creare oggetti. Definisce le proprietà (attributi) e i comportamenti (metodi) che gli oggetti creati da essa possiederanno. Visualizza esempio in JavaScript

Esempio Classe


    class Persona {
        constructor(nome, cognome) {
            this.nome = nome;
            this.cognome = cognome;
        }
        
        saluta() {
            console.log(`Ciao, mi chiamo ${this.nome} ${this.cognome}`);
        }
    }
    
    const persona1 = new Persona('Mario', 'Rossi');
    persona1.saluta(); // Output: Ciao, mi chiamo Mario Rossi

2. Oggetto

Un oggetto è un'istanza di una classe. È un'entità concreta che ha uno stato e un comportamento. Visualizza esempio in JavaScript

Esempio Oggetto


    const persona2 = new Persona('Luigi', 'Verdi');
    persona2.saluta(); // Output: Ciao, mi chiamo Luigi Verdi

3. Incapsulamento

L'incapsulamento è il concetto di nascondere i dettagli interni di un oggetto e permettere l'accesso solo attraverso metodi pubblici. Questo protegge l'integrità dei dati dell'oggetto. Visualizza esempio in JavaScript

Esempio Incapsulamento


    class ContoBancario {
        #saldo; // Variabile privata
        
        constructor(saldoIniziale) {
            this.#saldo = saldoIniziale;
        }
        
        deposita(importo) {
            if (importo > 0) {
                this.#saldo += importo;
            }
        }
        
        preleva(importo) {
            if (importo > 0 && importo <= this.#saldo) {
                this.#saldo -= importo;
            }
        }
        
        getSaldo() {
            return this.#saldo;
        }
    }
    
    const mioConto = new ContoBancario(100);
    mioConto.deposita(50);
    console.log(mioConto.getSaldo()); // Output: 150
    mioConto.preleva(20);
    console.log(mioConto.getSaldo()); // Output: 130

4. Ereditarietà

L'ereditarietà permette di creare una nuova classe che eredita le proprietà e i metodi di un'altra classe. Questo promuove il riuso del codice. Visualizza esempio in JavaScript

Esempio Ereditarietà


    class Studente extends Persona {
        constructor(nome, cognome, matricola) {
            super(nome, cognome); // Chiamata al costruttore della classe base
            this.matricola = matricola;
        }
        
        mostraMatricola() {
            console.log(`Matricola: ${this.matricola}`);
        }
    }
    
    const studente1 = new Studente('Giulia', 'Bianchi', '12345');
    studente1.saluta(); // Output: Ciao, mi chiamo Giulia Bianchi
    studente1.mostraMatricola(); // Output: Matricola: 12345

5. Polimorfismo

Il polimorfismo permette a oggetti di classi diverse di essere trattati come oggetti della stessa classe base. È un modo per utilizzare un'interfaccia comune per classi diverse. Visualizza esempio in JavaScript

Esempio Polimorfismo


    class Animale {
        parla() {
            console.log('L\'animale fa un suono');
        }
    }
    
    class Cane extends Animale {
        parla() {
            console.log('Il cane abbaia');
        }
    }
    
    class Gatto extends Animale {
        parla() {
            console.log('Il gatto miagola');
        }
    }
    
    function faiParlare(animale) {
        animale.parla();
    }
    
    const cane = new Cane();
    const gatto = new Gatto();
    
    faiParlare(cane); // Output: Il cane abbaia
    faiParlare(gatto); // Output: Il gatto miagola
